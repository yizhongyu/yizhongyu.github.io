Hadoop

Hadoop centos7
集群
发布日期:   2022-05-17
文章字数:   946

Hadoop 是一个开源的分布式计算和存储框架，由 Apache 基金会开发和维护。

Hadoop 为庞大的计算机集群提供可靠的、可伸缩的应用层计算和存储支持，它允许使用简单的编程模型跨计算机群集分布式处理大型数据集，并且支持在单台计算机到几千台计算机之间进行扩展。

Hadoop 使用 Java 开发，所以可以在多种不同硬件平台的计算机上部署和使用。其核心部件包括分布式文件系统 (Hadoop DFS，HDFS) 和 MapReduce。

安装Java

确认java安装成功
java -version
rpm -qa | grep jdk
卸载其他版本
yum -y remove java...		#或者
rpm -e --nodeps jdk...
上传jdk1.8rpm安装包
scp E:\app\Linux\jdk-18_linux-x64_bin.rpm root@192.168.200.99:/opt
执行安装jdk
rpm -ivh jdk-8u333-linux-x64.rpm

默认home路径：/usr/java/jdk1.8.0_333-amd64

添加环境变量文件到/etc/profile.d/
vim /etc/profile.d/java.sh
export JAVA_HOME=/usr/java/jdk1.8.0_333-amd64
export CLASSPATH=$JAVA_HOME/lib/
export PATH=$PATH:$JAVA_HOME/bin

安装Hadoop

上传Hadoop压缩包
scp E:\app\Linux\hadoop-2.10.1.tar.gz root@192.168.200.99:/opt

解压

mkdir /usr/hadoop
tar -zxvf hadoop-2.10.1.tar.gz -C /usr/hadoop

默认home路径：/usr/hadoop/hadoop-2.10.1

添加环境变量文件到/etc/profile.d/
vim /etc/profile.d/hadoop.sh
export HADOOP_HOME=/usr/hadoop/hadoop-2.10.1
export PATH=$PATH:$HADOOP_HOME/bin:$HADOOP_HOME/sbin
source /etc/profile

验证

hadoop version
whereis hdfs
创建HDFS数据目录
mkdir /var/HDFSdata

配置文件位置：/usr/hadoop/hadoop-2.10.1/etc/hadoop/

官方配置说明

为Hadoop提供Java解释器路径信息，主要解决远程访问Hadoop是Javahome无法继承的问题
vim /usr/hadoop/hadoop-2.10.1/etc/hadoop/hadoop-env.sh
# The java implementation to use.
export JAVA_HOME=/usr/java/default	（rpm安装路径）
为yarn任务，资源管理器提供Java运行环境
vim /usr/hadoop/hadoop-2.10.1/etc/hadoop/yarn-env.sh
# export JAVA_HOME=/home/y/libexec/jdk1.6.0/
if [ "$JAVA_HOME" != "" ]; then
  #echo "run java in $JAVA_HOME"
  JAVA_HOME=/usr/java/default		(这一步还是修改Javahome)
Hadoop核心文件配置(指定文件系统)
vim /usr/hadoop/hadoop-2.10.1/etc/hadoop/core-site.xml
<configuration>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://localhost:9000</value>	#修改localhost
    </property>
    <property>
        <name>hadoop.tmp.dir</name>
        <value>/var/HDFSdata</value>			#HDFS数据目录
    </property>
</configuration>
配置hdfs默认的数据存放策略
vim /usr/hadoop/hadoop-2.10.1/etc/hadoop/hdfs-site.xml
<configuration>
    <property>
        <name>dfs.replication</name>
        <value>2</value>						#hdfs副本数，大于1
    </property>
    <property>
        <name>dfs.namenode.secondary.http-address</name>
        <value>node3:50090</value>				#指定node3为secondary
    </property>
</configuration>
从模板拷贝mapreduce配置文件，配置任务调度策略
cp /usr/hadoop/hadoop-2.10.1/etc/hadoop/mapred-site.xml.template /usr/hadoop/hadoop-2.10.1/etc/hadoop/mapred-site.xml
vim /usr/hadoop/hadoop-2.10.1/etc/hadoop/mapred-site.xml
<configuration>
    <property>
        <name>mapreduce.framework.name</name>
        <value>yarn</value>
    </property>
</configuration>
配置yarn资源管理角色
vim /usr/hadoop/hadoop-2.10.1/etc/hadoop/yarn-site.xml
<configuration>
    <property>
        <name>yarn.nodemanager.aux-services</name>
        <value>mapreduce_shuffle</value>		#表示结构随机
    </property>
    <property>
        <name>yarn.resourcemanager.hostname</name>
        <value>node1</value>					#指定资源调度者
    </property>
</configuration>
配置datanode
vim /usr/hadoop/hadoop-2.10.1/etc/hadoop/slaves
node1
node2
node3
.....

主节点单独执行

格式化主节点hdfs
cd /usr/hadoop/hadoop-2.10.1/bin/
hdfs namenode-format
ls /var/HDFSdata/dfs/name/current/		#查看image
cd /usr/hadoop/hadoop-2.10.1/sbin/
start-dfs.sh
start-yarn.sh							#启动集群
jps										#查看
