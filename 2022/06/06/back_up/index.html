备份使用的工具有哪些?

本地备份: cp

远程备份: scp rsync sz rz

rsync 远程同步工具

  • 实现主机与主机之间的同步, 实现增量的同步. linux\windows\mac
  • PS: 如果将rsync当一个服务来使用,则需要知道 rsync 是个 c/s架构

rsync备份工具应用几种场景?

  • 推 客户端推送数据至服务端 电脑—>云盘
  • 拉 客户端拉取服务上的数据 电脑 <—- 云盘

rsync三大传输模式

yum install rsync -y

本地传输 类似于使用cp

Local:  rsync [OPTION...] SRC... [DEST]
rsync -avz oldxu.com /tmp/					#文件
rsync -avz /etc/ /tmp/						#目录
每次拷贝都是增量
1，如果有新增, 拷贝新增
2，如果修改历史拷贝过的文件, 则同步修改后的文件

远程传输 类似于scp,基于ssh协议

Pull: rsync [OPTION...] [USER@]HOST:SRC... [DEST]
rsync -avz root@172.16.1.31:/root/31.txt ./				#下载31服务器上/root/31.txt文件,至本地的当前目录,使用31的root身份
rsync -avz root@172.16.1.31:/opt/etc_test/ ./			#下载目录,以及目录下的所有内容至本地
rsync -avz root@172.16.1.31:/opt/etc_test ./`			#下载这个目录下的所有内容至本地
Push: rsync [OPTION...] SRC... [USER@]HOST:DEST
rsync -avz ./41.txt root@172.16.1.31:/opt/				#推送本地的文件或目录,至远程主机31的/opt目录下,使用的是31的root身份
参数 含义
-a 归档模式传输, 等于-tropgD
-v 详细模式输出, 打印速率, 文件数量等
-z 传输时进行压缩以提高效率
-r 递归传输目录及子目录，即目录下得所有目录都同样传输。
-t 保持文件时间信息
-o 保持文件属主信息
-p 保持文件权限
-g 保持文件属组信息
-l 保留软连接
-P 显示同步的过程及传输时的进度等信息
-D 保持设备文件信息
L 保留软连接指向的目标文件
e 使用的信道协议,指定替代rsh的shell程序 -e ‘ssh’
–exclude=PATTERN 指定排除不需要传输的文件模式
–exclude-from=file 文件名所在的目录文件
–bwlimit=100 限速传输
–partial 断点续传
–delete 让目标目录和源目录数据保持一致
