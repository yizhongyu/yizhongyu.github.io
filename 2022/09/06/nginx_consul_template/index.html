Nginx Consul consul-template keepalived
work
发布日期:   2022-09-06
文章字数:   1.8k

Consul_server是个小集群，Consul_agent是代理，nginx反向代理其他机器上面的服务，Consul_template实时更新nginx的代理策略，当有新的服务注册到consul时更新nginx配置文件，反之删除

keepalived保证D/E中的一台机器宕机后整个服务仍可用

规划

有A~F，六台服务器

A/B/C安装Consul_server

D/E安装nginx，Consul_template，keepalived

F安装Consul_agent

flowchart TB
    idX>虚拟IP192.168.200.111]==keepalived===idD[服务器D192.168.200.94]
    idX--keepalived---idE[服务器E192.168.200.95]
    idA(服务器A192.168.200.91)---idB(服务器B192.168.200.92)---idC(服务器C192.168.200.93)
    	subgraph Consul_server
    	idA & idB & idC
    	end
    	subgraph nginx
    	idD & idE
    	end
    	idD & idE --consul-template--- Consul_server
Consul_server --consul_agent--- idF(服务器F192.168.200.96)

软件安装包放在/opt目录consul下载，consul-template下载，基于centos7.9_2207

需要统一创建的目录

mkdir -p /usr/local/gridknow/consul/consul.d
mkdir -p /usr/local/gridknow/consul/logs
mkdir -p /usr/local/gridknow/consul/template/data
mkdir -p /usr/local/gridknow/consul/template/logs/

一：Consul安装

安装
unzip /opt/consul_1.13.1_linux_amd64.zip -d /usr/local/gridknow/consul
ln -s /usr/local/gridknow/consul/consul /usr/local/bin

验证consul version

1，Consul server端 -bind=IP执行机器本机IP

节点一启动consul_server

consul agent -server -bind=192.168.200.91 -client=0.0.0.0 -ui -bootstrap-expect 3 -data-dir=/usr/local/gridknow/consul/data -config-dir=/usr/local/gridknow/consul/consul.d -datacenter=gridknow &

节点二启动consul_server

consul agent -server -bind=192.168.200.92 -client=0.0.0.0 -ui -bootstrap-expect 3 -data-dir=/usr/local/gridknow/consul/data -config-dir=/usr/local/gridknow/consul/consul.d -datacenter=gridknow &

节点三启动consul_server

consul agent -server -bind=192.168.200.93 -client=0.0.0.0 -ui -bootstrap-expect 3 -data-dir=/usr/local/gridknow/consul/data -config-dir=/usr/local/gridknow/consul/consul.d -datacenter=gridknow &
参数含义
-server		#以服务端身份启动(注册中心)
-bind		#标识绑定到哪个IP
-client		#指定客户端访问IP，0.0.0.0表示不限制客户端IP
-ui			#打开web界面
-bootstrap-expect		#集群节点至少三台，官方建议3台或者5台(奇数选举)
-data-dir	#数据存储目录
-node		#定义节点名称，默认节点主机名(不可同名)
-datacenter	#定义数据中心名称，默认dc1(唯一)
-config-dir=/usr/local/gridknow/consul/consul.d/ #加这个参数表示加载这里的服务
consul_server启动配置文件

vim /usr/local/gridknow/consul/consul.d/consul_server.hcl

server = true
ui = true
bootstrap_expect = 3
data_dir = "/usr/local/gridknow/consul/data"
log_file = "/usr/local/gridknow/consul/logs/"
datacenter = "gridknow"
enable_script_checks = true
client_addr = "0.0.0.0"
bind_addr ="192.168.200.91"

2，Consul agent端

client模式启动consul，并加入集群 -bind=IP执行机器本机IP

nohup consul agent -bind=192.168.200.96 -data-dir=/usr/local/gridknow/consul/data -config-dir=/usr/local/gridknow/consul/consul.d -datacenter=gridknow -join 192.168.200.91 >> /usr/local/gridknow/consul/logs/consul.log 2>&1 &
命令参数说明
  • -join：加入一个已经启动的agent的ip地址，可以多次指定多个agent的地址。如果consul不能加入任何指定的地址中，则agent会启动失败，默认agent启动时不会加入任何节点。
consul_agent启动配置文件

vim /usr/local/gridknow/consul/consul.d/consul_agent.hcl

data_dir = "/usr/local/gridknow/consul/data"
log_file = "/usr/local/gridknow/consul/logs/"
datacenter = "gridknow"
enable_script_checks = true
client_addr = "0.0.0.0"
bind_addr ="192.168.200.96"
retry_join = ["192.168.200.91"]

3，节点加入集群

consul join 192.168.200.91
查看集群命令
consul members
注册服务

/usr/local/gridknow/consul/consul.d/web.json

{
  "service": {
    "name": "web",
    "tags": [
      "rails"
    ],
    "port": 80
  }
}

可在代理节点上执行

consul services register /usr/local/gridknow/consul/consul.d/web.json
consul启动命令

consul agent -config-dir=/usr/local/gridknow/consul/consul.d/

4，编写开机脚本

需要consul.d/目录有配置文件

vim /usr/lib/systemd/system/consul.service

[Unit]
Description=consul
After=network.target
[Service]
Type=simple
User=root
ExecStart=/usr/local/bin/consul agent -config-dir /usr/local/gridknow/consul/consul.d/
Restart=on-failure
[Install]
WantedBy=multi-user.target
systemctl start consul.service
systemctl enable consul.service

日志

tail /usr/local/gridknow/consul/logs/consulXXX

二：Nginx安装

在线安装
yum install yum-utils
vim /etc/yum.repos.d/nginx.repo
[nginx-stable]
name=nginx stable repo
baseurl=http://nginx.org/packages/centos/$releasever/$basearch/
gpgcheck=1
enabled=1
gpgkey=https://nginx.org/keys/nginx_signing.key
module_hotfixes=true

[nginx-mainline]
name=nginx mainline repo
baseurl=http://nginx.org/packages/mainline/centos/$releasever/$basearch/
gpgcheck=1
enabled=0
gpgkey=https://nginx.org/keys/nginx_signing.key
module_hotfixes=true

默认情况下，使用稳定nginx包的存储库。如果要使用主线 nginx 包，请运行以下命令：

yum-config-manager --enable nginx-mainline

要安装 nginx，请运行以下命令：

yum install nginx
systemctl enable nginx

三：consul-template

安装
unzip /opt/consul-template_0.29.2_linux_amd64.zip -d /usr/local/gridknow/consul/template/
ln -s /usr/local/gridknow/consul/template/consul-template /usr/local/bin/

验证consul-template -v

新建Consul-Template模板

vim /usr/local/gridknow/consul/template/data/nginx.conf.ct

{{range services}} {{$name := .Name}} {{$service := service .Name}}
upstream {{$name}} {
zone upstream-{{$name}} 64k;
{{range $service}}server {{.Address}}:{{.Port}} max_fails=3 fail_timeout=60 weight=1;
{{else}}server 127.0.0.1:65535; # force a 502{{end}}
} {{end}}
server {
listen 8088;
{{range services}} {{$name := .Name}}
location /{{$name}}/ {
proxy_pass http://{{$name}}/;
}
{{end}}
}
生成nginx配置文件

不启动consul_agent， -consul-addr 127.0.0.1:8500的IP需要填集群IP

调用模板文件生成查询结果

consul-template -consul-addr 192.168.200.91:8500 -template="/usr/local/gridknow/consul/template/data/nginx.conf.ct:/etc/nginx/conf.d/upstream.conf" -once

如果想生成Nginx配置文件后自动加载配置，可以这样：

consul-template -consul-addr 127.0.0.1:8500 -template="/usr/local/gridknow/consul/template/data/nginx.conf.ct:/etc/nginx/conf.d/upstream.conf:/usr/sbin/nginx -s reload" -once

指定日志输出到相应目录，并设置后台启动

nohup consul-template -consul-addr 127.0.0.1:8500 -template="/usr/local/gridknow/consul/template/data/nginx.conf.ct:/etc/nginx/conf.d/upstream.conf:/usr/sbin/nginx -s reload" >> /usr/local/gridknow/consul/template/logs/consul-template.log 2>&1 &
编写启动脚本

vim /usr/local/gridknow/consul/template/start.sh

#!/bin/bash
nohup consul-template \
-consul-addr 192.168.200.91:8500 \
-template="/usr/local/gridknow/consul/template/data/nginx.conf.ct:/etc/nginx/conf.d/upstream.conf:/usr/sbin/nginx -s reload" \
>> /usr/local/gridknow/consul/template/logs/consul-template.log 2>&1

chmod +x /usr/local/gridknow/consul/template/start.sh

注册为Systemd服务

vim /usr/lib/systemd/system/consul-template.service

[Unit]
Description=consul-template
After=network.target
    
[Service]
ExecStart=/usr/local/gridknow/consul/template/start.sh
KillSignal=SIGTERM
    
[Install]
WantedBy=multi-user.target
systemctl start consul-template
systemctl enable consul-template

日志

tail /usr/local/gridknow/consul/template/logs/consul-template.log

四：keepalived

虚拟IP代理俩台Nginx服务器

yum install keepalived
systemctl enable keepalived
keepalived配置

改配置文件(将默认的eth0网卡改为自己的网卡名,绑定当前网卡)

sed -i "s/eth0/ens192/g" /etc/keepalived/keepalived.conf

设置主备(改其中一个为BACKUP并调低优先级)

sed -i "s/MASTER/BACKUP/g" /etc/keepalived/keepalived.conf
sed -i "s/priority 100/priority 90/g" /etc/keepalived/keepalived.conf

改虚拟ip

! Configuration File for keepalived

global_defs {
   router_id LVS_DEVEL
}

vrrp_instance VI_1 {
    state MASTER
    interface ens192
    virtual_router_id 51
    priority 100
    advert_int 1
    authentication {
        auth_type PASS
        auth_pass 1111
    }
    virtual_ipaddress {
        192.168.200.111
    }
}

keepalived与nginx联动

vrrp_script check_nginx {					#定义一个策略,策略名为"check_nginx"
	script "/etc/keepalived/chech_nginx.sh"	#执行一次/etc/keepalived/check_nginx.sh脚本文件
	interval 5								#每隔5秒钟
	weight -15								#如果发现nginx启动失败了,将当前的机器的priority-15
}
    track_script {							#指定keepalived跟踪策略为 "check_nginx"
    	check_nginx
    }

Keepalived_MASTER节点标准配置文件

! Configuration File for keepalived

global_defs {
   router_id LVS_DEVEL
}

vrrp_script check_nginx {
	script "/etc/keepalived/chech_nginx.sh"
	interval 5
	weight -15
}

vrrp_instance VI_1 {
    state MASTER
    interface ens192
    virtual_router_id 51
    priority 100
    advert_int 1
    authentication {
        auth_type PASS
        auth_pass 1111
    }
    track_script {
    	check_nginx
    	}
    virtual_ipaddress {
        192.168.200.111
    }
}

监测脚本vim /etc/keepalived/chech_nginx.sh

#!/bin/bash
A=`ps -C nginx --no-header |wc -l`        
if [ $A -eq 0 ];then                            
	/usr/sbin/nginx
	sleep 5
	if [ `ps -C nginx --no-header |wc -l` -eq 0 ];then
		killall keepalived
	fi
fi

chmod +x /etc/keepalived/chech_nginx.sh
