kubernetes安装

docker Kubernetes
容器
发布日期:   2022-09-14
文章字数:   756

裸机安装，需要至少俩台服务器

  1. master节点需要的组件
  • docker
  • kubectl：集群命令行交互工具
  • kubeadm：集群初始化工具
  1. node节点需要的组件
  • docker
  • kubelet：管理pod的容器，确保他们健康稳定的运行
  • Kube-proxy： 网络代理，负责网络相关工作

前置条件：

设置hostname，添加域名映射，做好节点免密，关闭防火墙和selinux，关闭swap，时间同步

sed -i 's/.*swap.*/#&/' /etc/fstab			#关闭swap

添加k8s安装源和docker安装源

cat <<EOF > /etc/yum.repos.d/kubernetes.repo
[kubernetes]
name=Kubernetes
baseurl=http://mirrors.aliyun.com/kubernetes/yum/repos/kubernetes-el7-x86_64
enabled=1
gpgcheck=1
repo_gpgcheck=0
gpgkey=http://mirrors.aliyun.com/kubernetes/yum/doc/yum-key.gpg http://mirrors.aliyun.com/kubernetes/yum/doc/rpm-package-key.gpg
EOF

yum makecache

yum-config-manager \
    --add-repo \
    https://download.docker.com/linux/centos/docker-ce.repo

yum-config-manager --add-repo http://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo

如果报错 -bash: yum-config-manager: command not found 则运行 yum install -y yum-utils

安装k8s组件

yum list kubelet --showduplicates | tail -n 10 查看版本号

yum install -y kubelet-1.16.15 kubeadm-1.16.15 kubectl-1.16.15 docker-ce-18.09.9-3.el7
yum install -y kubelet-1.21.14 kubeadm-1.21.14 kubectl-1.21.14 docker-ce
yum install -y kubelet-1.23.1 kubeadm-1.23.1 kubectl-1.23.1 docker-ce

Kubectl 管理 Kubernetes 集群命令行工具

Kubeadm 是一个快捷搭建kubernetes(k8s)的安装工具,它提供了kubeadm init 以及 kubeadm join这两个命令来快速创建kubernetes集群

kubelet 是在每个 Node 节点上运行的主要 “节点代理”

启动 docker、kubelet 服务

systemctl enable kubelet && systemctl start kubelet		#开机自启并启动kubelet服务
systemctl enable docker && systemctl start docker		#开机自启并启动docker服务

修改 docker 配置

官方建议修改 docker Cgroup Driver 为 systemd

cat <<EOF > /etc/docker/daemon.json
{
  "exec-opts": ["native.cgroupdriver=systemd"],
  "registry-mirrors": ["https://m24eqnhh.mirror.aliyuncs.com"]
}
EOF

systemctl daemon-reload && systemctl restart docker

重启完之后可使用docker info查看是否修改成功

kubeadm 初始化集群（主节点执行）

kubeadm init \
--apiserver-advertise-address=192.168.200.98 \
--image-repository registry.aliyuncs.com/google_containers \
--kubernetes-version=v1.21.14 \
--pod-network-cidr=10.244.0.0/16 \
--service-cidr=10.96.0.0/16

节点添加认证

方法一：添加环境变量（主节点执行）

echo "export KUBECONFIG=/etc/kubernetes/admin.conf" >> /etc/profile
source /etc/profile

方法二：修改认证文件访问权限（主节点执行）

mkdir -p /root/.kube
cp -i /etc/kubernetes/admin.conf /root/.kube/config

node节点添加认证（从节点执行）

mkdir -p /root/.kube
scp master:/etc/kubernetes/admin.conf /root/.kube/config

节点加入集群成功，使用 kubectl get nodes 查看节点信息，发现都是 NotReady，需要在主节点安装网络插件 Flannel 或 Calicos

安装网络插件

kubectl apply -f http://projectcalico.docs.tigera.io/manifests/calico.yaml

kubectl get pod -o wide -n kube-system 查看所有系统的 pod

kubectl get pods -n kube-system 查看 容器运行状态
